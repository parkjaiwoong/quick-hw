# 예상 배송 시간 초과 시 일반적 처리 방법

고객이 선택한 예상 시간(급송 30분, 기본 3시간)을 넘겨 배송이 완료되는 경우, 플랫폼에서 흔히 쓰는 대응 방식을 정리했습니다.

## 1. **사전 예방 (기사 앱)**

- 배송 진행 중 화면에 **예상 완료 시각**과 **"배송 완료 버튼을 눌러 주세요"** 안내 노출
- 예상 시각이 지나면 **"예상 시간이 초과되었습니다"** 배너로 알림
- 완료 후에도 **"예상 시간을 N분 초과하여 완료되었습니다"** 문구로 인지시키기

→ 본 프로젝트에서는 `ExpectedTimeBanner`, `ExceededCompleteMessage`로 구현됨.

## 2. **데이터 관리 (관리자)**

- **예상시간 초과 건만 모아서 조회**하는 화면 제공  
  - 관리자 > 주문 & 연결 로그 > **예상시간 초과 배송** (`/admin/late-deliveries`)
- 초과 건에 대해 **초과 분수, 고객/기사, 요금** 등 확인 가능
- 기사별·기간별 집계로 **반복 초과** 여부 파악 후 교육·평가에 활용

## 3. **운영 정책에서의 활용**

| 구분 | 일반적 처리 |
|------|-------------|
| **고객 보상** | 초과 시간이 길면 할인 쿠폰, 포인트 적립 등으로 사과·보상 (정책에 따라) |
| **기사 평가** | 초과 비율·횟수를 KPI로 반영하거나, 반복 시 교육·패널티 정책 수립 |
| **정산** | 대부분 정산은 그대로 두고, 별도 보상/패널티 정책으로 연동 |
| **CS** | 고객 문의 시 "예상시간 초과 배송" 목록으로 해당 건을 빠르게 찾아 대응 |

## 4. **추가로 할 수 있는 것**

- **실시간 알림**: 예상 시각 N분 전에 기사에게 푸시로 "곧 예상 시간입니다" 안내
- **고객 알림**: "배송이 예상보다 지연되고 있습니다" 메시지 (기사 확인 후 발송 등)
- **리포트**: 주간/월간 예상시간 준수율 리포트로 운영 개선

이 문서는 참고용이며, 실제 보상·패널티·KPI 반영 방식은 서비스 정책에 맞게 결정하면 됩니다.
